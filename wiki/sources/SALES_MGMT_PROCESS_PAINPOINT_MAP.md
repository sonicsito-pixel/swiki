## 2. 업무 단위별 상세 Flow 및 Pain Point 매핑

### [P1] 매장 기준정보 관리 프로세스
- **Flow**: 신규 매장 확보 -> 사업자 등록 -> 시스템 매장 코드 생성 -> 매장별 계약/수수료 정보 입력
- **Pain Point Mapping**:
    - **[P1-1] 데이터 파편화**: 사업자 번호 변경 시 신규 코드를 따야 하므로 과거 매출 이력과 단절됨.
    - **[P1-2] 관리 책임 불분명**: 영업/회계/인사 부서가 한 화면에서 입력하여 데이터 오염 발생.
- **To-Be Flow**: 고유 매장 ID 자동 채번 -> 부서별 전용 입력 탭 활성화 -> SCD 엔진이 속성 변경 이력 자동 기록.

### [P2] 공급분 매입 프로세스 (신규 보강)
- **Flow**: 발주서 생성 -> 물류 입고 -> 매입 전표 수기 생성 -> 거래처 확정
- **Pain Point Mapping**:
    - **[P2-1] 중복 입력**: 물류 입고 데이터가 있음에도 영업 담당자가 다시 매입 전표를 생성함.
    - **[P2-2] 프로세스 복잡성**: 단순 관리용 가상 매입 단계가 많아 데이터 정합성 확인이 어려움.
- **To-Be Flow**: 물류 입고 시 매입 자동 확정(Auto-Matching) -> 가상 트랜잭션 단계 축소 -> 매입/매출 이익 실시간 대시보드.

### [P3] 매출 및 정산 프로세스
- **Flow**: 일 매출 발생 -> 유통망 EDI 수집 -> 카드 입금 대조 -> 중간관리자 공제 계산 -> 정산 확정
- **Pain Point Mapping**:
    - **[P3-1] 엑셀 대조 지옥**: 수천 개의 매장 매출을 유통망 데이터와 엑셀로 VLOOKUP 대조함.
    - **[P3-2] 정산 오류**: 매장별 복잡한 공제 항목(물류비, 소모품 등)을 수기 계산하여 누락 발생.
- **To-Be Flow**: EDI 데이터 자동 수집 -> 시스템 내 3-Way 자동 매칭 -> 공제 항목 템플릿 자동 적용 -> 전자세금계산서 역발행.

### [P4] 재고 수불 프로세스 (Hub-Spoke 중심)
- **Flow**: 거점 매장 입고 -> 종속 매장 재고 이동 -> 판매 -> 매장 간 재고 정산
- **Pain Point Mapping**:
    - **[P4-1] 이동 중 재고 분실**: 매장 간 이동(RT) 중인 재고의 현재 위치와 책임 소재가 불분명함.
    - **[P4-2] 가상창고 남용**: 시스템 제약으로 YA센터 등 불필요한 가상 창고를 만들어 데이터 꼬임 발생.
- **To-Be Flow**: RT 발송 시 '이동 중 재고' 가시화 -> 입고 확정 시 책임 전가 -> 가상 창고 폐쇄 및 실제 물리 위치 기반 수불.

### [P5] 채권 및 리스크 관리 프로세스
- **Flow**: CMS 입금 수집 -> 채권 잔액 상계 -> 미수금 확인 -> 영업 담당자 독촉
- **Pain Point Mapping**:
    - **[P5-1] 대응 실기(Timing)**: 채권 파악이 1~2일 지연되어 미수금이 많은 매장에 계속 물건이 나감.
    - **[P5-2] 수기 제어**: 미수 매장 판매 중지를 담당자가 전화를 하거나 수동으로 설정해야 함.
- **To-Be Flow**: 입금 시 실시간 채권 상계 -> 임계치 초과 시 시스템 자동 판매 Blocking -> 리스크 스코어 알림 알림.
