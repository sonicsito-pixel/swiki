# 영업관리 PI 분석 결과 종합 보고서 (RFP 기초안)

**작성일**: 2026-04-24
**작성자**: Gemini CLI
**대상**: 차세대 패션 ERP(F-ONE) 프로젝트 추진팀

---

## 1. 개요
본 보고서는 차세대 패션관리 시스템 구축을 위한 영업관리 PI(Process Innovation) 분석의 최종 결과물임. 5대 핵심 업무 영역(매장, 매출, 수불, 채권, 증빙)의 AS-IS 프로세스를 분석하고, 외부 개발업체에 요구할 차세대 시스템의 핵심 기능을 정의함.

## 2. 현행 업무 진단 및 문제점 요약
현재 영업관리 업무는 데이터의 단절과 수기 엑셀 작업에 과도하게 의존하고 있음.
- **Data Fragmentation**: 사업자 중심의 매장 관리로 인한 데이터 이력 단절.
- **Manual Labor**: 유통망 매출 수집, 입금 대조, 정산 공제 등의 업무가 시스템 밖 엑셀에서 수행됨.
- **Control Gap**: 입금 데이터(CMS)는 수집되나 실시간 채권 상계 및 판매 제어와 유기적으로 연동되지 않음.
- **In-Transit Blind Spot**: 매장 간 재고 이동 시 위치 추적이 불투명하고 가상 창고 오남용 발생.

## 3. 차세대 시스템(TO-BE) 핵심 요구사항 (RFP 반영 항목)

### 3.1 [기준정보] Universal 매장 마스터 및 SCD 엔진
- 사업자 변경과 무관한 고유 매장코드 체계 수립.
- 시점별 속성 변화를 기록하는 SCD(Slowly Changing Dimension) 이력 관리.
- 부서별 전용 입력 탭(Tab) 기반 데이터 거버넌스 강화.

### 3.2 [정산/증빙] Zero-Manual 자동 정산 플랫폼
- 주요 유통망 EDI 및 VAN 데이터 자동 수집 인터페이스 구축.
- 3-Way(ERP-유통망-VAN) 자동 매칭 엔진 및 예외 관리 대시보드.
- Trustbill API 연동을 통한 세금계산서 자동 역발행 및 이미지 통합 조회.

### 3.3 [물류/수불] Hub-Spoke 실시간 수불 시스템
- 거점-종속 관계 정의를 통한 실시간 재고 공유 및 자동 매출 배분 로직.
- '이동 중 재고' 실시간 위치 추적 및 미확정 건 자동 Alert.
- 매장 현장 완결형 자가소모 처리 및 회계 전표 자동 연동.

### 3.4 [리스크/채권] 액티브 리스크 모니터링 체계
- CMS 입금 데이터의 실시간 채권 상계 및 가상계좌 1:1 매핑 고도화.
- 미수 임계치 초과 시 시스템 자동 판매 제어(Auto-Blocking) 로직.
- 실시간 채권 연령 분석(Aging) 및 리스크 스코어링 대시보드.

## 4. 전략적 로드맵
- **단기**: 매장 ID 자동 채번, 매출 목표 히스토리 DB, 미사용 마스터 정리.
- **중기**: WMS-FONE 창고 일원화, PLM-RFID 연동, 자가소모 현장 처리.
- **장기**: 브랜드 코드 체계 확장(2자리), AI 기반 최적 재고 배분, 레거시 현대화.

## 5. 결론
차세대 시스템은 단순히 현행 기능을 웹으로 옮기는 것이 아니라, **'데이터 기반의 자동화된 의사결정 지원'**이 가능한 플랫폼이 되어야 함. 본 보고서의 요구사항을 기반으로 RFP를 작성하여 외부 개발업체의 제안 범위를 명확히 할 것을 권고함.
