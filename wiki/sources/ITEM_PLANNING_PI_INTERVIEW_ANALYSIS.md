# [PI 분석] 상품기획(Item Planning) 현업 인터뷰 결과 및 개선 방향

**일시:** 2026년 4월 30일 (목)
**대상:** 상품기획팀 현업 담당자 및 IT기획팀
**문서번호:** PI-ITEM-20260430-01

---

## 1. 개요
본 문서는 2026년 4월 30일 진행된 상품기획팀 인터뷰 내용을 바탕으로, 차세대 패션관리 시스템(FA-ONE) 구축을 위한 핵심 페인포인트(Pain Point)를 4단계 표준 분석 프레임워크([As-Is] - [To-Be] - [Gap] - [해결방안])에 따라 정리한 보고서입니다.

---

## 2. 핵심 이슈별 PI 분석

### [이슈 1] 기획-생산-운영 간 시스템 단절 (End-to-End 통합 부재)
*   **As-Is:** Centric PLM(기획), F-ONE(ERP), S-ONE(소싱) 등 시스템이 분절되어 운영됨. 각 단계별 데이터 연동이 원활하지 않아 MD가 엑셀을 통해 수기로 데이터를 매핑하고 재입력하는 비효율 발생.
*   **To-Be:** 기획부터 판매까지 데이터가 끊김 없이 흐르는 통합 파이프라인 구축. RFID 및 향후 확장성을 고려한 데이터 아키텍처 수립.
*   **Gap:** 시스템 간 데이터 인터페이스 규격 미비 및 마스터 데이터 불일치.
*   **해결방안 (RFP):** PLM-ERP-Sourcing 시스템 간 실시간 API 연동 및 데이터 동기화 로직 구현. 전사 통합 데이터 흐름(End-to-End) 가시성 확보.

### [이슈 2] 작업지시서(PO) 히스토리 관리 및 데이터화 미흡
*   **As-Is:** 현재 F-ONE의 작지는 엑셀로 작성된 PDF나 이미지를 단순 업로드하는 방식. 도식화나 스펙 수정 시 변경 이력(History) 추적이 불가능하며 데이터로서의 활용도가 낮음.
*   **To-Be:** 시스템 내에서 작지 생성, 수정, 확정 프로세스가 관리되며 버전별 히스토리 트래킹 및 데이터 자산화 실현.
*   **Gap:** 작지 관리 기능이 '파일 저장소' 수준에 머물러 있으며, 구조화된 데이터 입력 필드 부재.
*   **해결방안 (RFP):** 구조화된 작지 관리 모듈 개발. 버전 관리(Version Control), 수정 이력 로그, 도식화 및 스펙 데이터의 DB화. 향후 생성형 AI 연동을 위한 확장 인터페이스 확보.

### [이슈 3] AI 분석을 위한 상품 속성 및 기준정보 표준화 부재
*   **As-Is:** 브랜드(FS vs 탑텐), 사업부, 담당자마다 상품 속성(핏, 넥라인, 소재감 등) 정의 기준이 다름. 주관적 용어 사용으로 인해 데이터 분석(MOFL 등 AI 연동) 시 신뢰도 저하.
*   **To-Be:** 전사 공통 상품 속성 체계(Tagging System) 구축. AI가 즉시 학습 및 분석 가능한 수준의 표준화된 마스터 데이터 운영.
*   **Gap:** 전사 차원의 마스터 데이터 거버넌스(Governance) 및 표준 가이드라인 부재.
*   **해결방안 (RFP):** 상품 속성 마스터(Item Attribute Master) 재정의 및 표준화. 브랜드별 특성을 수용하면서도 전사 통합 분석이 가능한 계층형 속성 체계 구축.

### [이슈 4] 상품 속성 입력의 높은 업무 부하 (Manual Entry)
*   **As-Is:** 세분화된 상품 속성을 MD가 일일이 시스템에 수기 입력해야 함. 과도한 업무량으로 인해 데이터 입력 누락이나 오류 빈번.
*   **To-Be:** AI 이미지 인식 기술을 활용하여 제품 사진(Lookbook, 온라인용) 업로드 시 형태, 색상, 실루엣 등 속성을 자동 분류 및 제안.
*   **Gap:** 단순 반복적인 데이터 입력 업무의 자동화 기술 적용 부재.
*   **해결방안 (RFP):** AI 기반 '이미지 속성 자동 추출 및 분류' 기능 도입. 이미지 분석 결과와 시스템 속성 필드 매핑 자동화.

### [이슈 5] 파편화된 데이터로 인한 분석 비효율 (Excel 의존도)
*   **As-Is:** 가격 인하 이력, 사이즈별 판매량 등 다각도 분석을 위해 여러 화면에서 데이터를 다운로드하여 엑셀(VLOOKUP)로 재가공함. 실시간 의사결정 지연.
*   **To-Be:** 사용자 중심의 통합 대시보드 및 자연어 기반 쿼리(Prompt)를 통한 유연한 데이터 분석 환경 제공.
*   **Gap:** 사용자 정의형 보고서 기능 부재 및 분산된 UI 구조.
*   **해결방안 (RFP):** 통합 BI 대시보드 구축. 주요 지표(KPI) 시각화 및 사용자가 직접 조건을 조합하여 리포트를 생성할 수 있는 Ad-hoc Query 기능 강화.

## 3. 향후 조치 사항 (Next Steps)
1.  **수기 양식 수집:** 현업에서 실제 사용 중인 통합 스케줄 시트, 작지, 속성 정의서 등 엑셀 파일 샘플 확보 (IT기획팀).
2.  **데이터 모델링:** 이슈 3번(표준화)을 해결하기 위한 전사 통합 상품 데이터 스키마 설계 착수.
3.  **심층 인터뷰:** 브랜드별 특수성 반영을 위한 MD별 2차 심층 인터뷰 실시.
4.  **RFP 반영:** 위 5가지 해결방안을 차세대 ERP 구축 RFP의 핵심 요구사항으로 구체화.
