# AI 기반 경영보고 자동화 프로젝트 단계별 과업 상세 정의서

본 문서는 'AI 기반 경영보고 자동화 단계적 기획안'을 바탕으로, 프로젝트의 성공적인 수행을 위한 순차적 진행 단계와 단계별 핵심 과업을 기술합니다.

## 1. 프로젝트 진행 단계 (Phase Roadmap)

본 프로젝트는 데이터 표준화에서 시작하여 분석 자동화, 최종적으로는 지능형 의사결정 지원 단계로 진화합니다.

- **Phase 1: Narrative Automation (보고서 서술 자동화)**
- **Phase 2: Diagnostic AI (원인 분석 자동화)**
- **Phase 3: Agentic AI (자율 탐색 및 시뮬레이션)**

---

## 2. 단계별 핵심 과업 및 성공 전략

### Phase 1: Narrative Automation (준비 및 보고 자동화)
*목표: 수기 작업 제거 및 AI 기반 요약 문장 자동 생성*

| 구분 | 핵심 과업 (Tasks) | 성공 포인트 (Key Factors) |
| :--- | :--- | :--- |
| **KPI 표준화** | - 전사 공통 KPI Dictionary 정의 (계산식, 담당자, 임계값)<br>- 보고 목적별 Narrative Hint(서술 가이드) 정의 | 부서 간 KPI 해석의 차이를 완전히 제거해야 함 |
| **데이터 파이프라인** | - Data Mart 연동용 SQL Template 작성<br>- Semantic KPI JSON 생성용 Python ETL 스크립트 개발 | AI가 오해하지 않도록 '의미가 부여된 데이터' 구조 설계 |
| **AI 엔진 구축** | - 임계값 기반 이상 탐지 Rule Engine 구현<br>- 표준 프롬프트 템플릿(Prompt Template) 설계 및 튜닝 | AI가 절대 계산을 수행하지 않고 '해설'만 하도록 제어 |
| **자동화 도구** | - PPT 자동 생성 라이브러리 연동<br>- 주간 보고 자동 발송 스케줄러 구축 | MVP(주간 매출보고)의 정기적인 발행 및 안정성 확보 |

### Phase 2: Diagnostic AI (분석 자동화)
*목표: 매출 등 KPI 변화의 근본 원인(Root Cause) 자동 도출*

| 구분 | 핵심 과업 (Tasks) | 성공 포인트 (Key Factors) |
| :--- | :--- | :--- |
| **인과관계 모델링** | - KPI Driver Tree 구조화 (매출 ↔ 객수 ↔ 객단가 등)<br>- Driver 간 영향도(Impact) 및 우선순위 매핑 테이블 구축 | 현업 MD/영업 전문가의 업무 지식을 데이터화하는 과정 |
| **오케스트레이션** | - 이상 탐지 시 연관 Driver를 자동 조회하는 엔진 개발<br>- 다차원 분석을 위한 SQL 파라미터 자동 생성 | 원인 분석의 경로(Path)를 Rule 기반으로 명확히 정의 |
| **RCA 자동화** | - Root Cause Analysis 결과의 Narrative 생성<br>- Tableau MCP(Natural Language Query) 연동 검토 | "왜?"라는 질문에 대해 데이터에 근거한 답변 생성 |

### Phase 3: Agentic AI (의사결정 지원)
*목표: AI 에이전트가 스스로 가설을 세우고 시뮬레이션 수행*

| 구분 | 핵심 과업 (Tasks) | 성공 포인트 (Key Factors) |
| :--- | :--- | :--- |
| **지식 그래프 구축** | - Neo4j 기반 KPI 지식 그래프(Knowledge Graph) 설계<br>- 비정형 데이터(인터뷰, 과거 보고서)의 Graph RAG 연동 | 데이터 간의 복잡한 의존 관계를 그래프로 시각화 및 관리 |
| **에이전트 제어** | - LangGraph 기반의 상태 관리형 AI Agent 워크플로우 설계<br>- Human-in-the-loop (중요 의사결정 시 사람 승인) 구조 구현 | AI의 자율성을 통제할 수 있는 안전 장치(Safety Rail) 마련 |
| **시뮬레이션** | - 할인율 변경, 채널 조정 등 시나리오별 실적 예측 모델 연동<br>- 부서별(재무/재고/마케팅) 멀티 에이전트 협업 체계 구축 | 단순 보고를 넘어 실질적인 '전략 제안' 수준의 결과 도출 |

---

## 3. 공통 기반 과업 (Foundation & Governance)

프로젝트 전반에 걸쳐 지속적으로 관리되어야 할 기반 과업입니다.

1.  **품질 검증 (Quality Gate):** AI가 생성한 Narrative의 정확도를 Tableau 수치와 대조하여 상시 모니터링하는 검증 프로세스 운영.
2.  **보안 및 권한:** PII(개인정보) 마스킹, API 키 관리 및 기업 기밀 데이터의 외부 유출 방지 아키텍처 준수.
3.  **Audit Trail:** AI가 내린 모든 결론에 대해 '어떤 데이터(SQL)'와 '어떤 논리(Prompt)'를 사용했는지 추적 가능한 로그 관리.
4.  **변화 관리 (Change Management):** 자동 생성 보고서에 대한 실무자의 불신을 해소하기 위해 기존 방식과 병행 운영하며 피드백 수렴.
