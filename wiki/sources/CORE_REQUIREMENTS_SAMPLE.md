# [FA-ONE] 차세대 ERP 핵심 요구사항 정의서 (Sample)

**프로젝트:** 차세대 패션관리 시스템(FA-ONE) 구축 PI
**작성일:** 2026-05-07
**작성자:** Gemini CLI (Requirement Analyst Mode)

---

## 1. 영업관리 (Sales Management) 핵심 요구사항

### [REQ-SD-01] 매장코드 기반 통합 이력 관리 시스템
- **As-Is:** 사업자번호가 Primary Key로 설정되어 점주 변경 시 과거 데이터 연계 불가.
- **To-Be:** 매장 고유 코드를 기반으로 점주/사업자 정보의 변경 이력(SCD Type 2)을 관리하여 데이터 연속성 확보.
- **사용자 스토리:** 
    - "영업 담당자로서 매장 점주가 바뀌더라도 해당 위치 매장의 과거 3개년 매출 추이를 즉시 조회하여 영업 전략을 수립하고 싶다."
- **수용 기준 (AC):**
    1. 신규 매장 등록 시 시스템 자동 채번 기능을 통한 '매장코드' 생성 필수.
    2. 매장 마스터 하위에 '사업자 정보' 탭을 두어 유효기간(Start/End Date) 기반의 이력 관리 기능 구현.
    3. 모든 매출/재고 현황판에서 '매장코드'로 조회 시 하위 모든 사업자 이력 데이터 합산 기능 제공.

### [REQ-SD-02] 유통망 매출 및 입금 자동 매칭 엔진 (Zero-Manual)
- **As-Is:** 백화점/유통망 매출 데이터를 엑셀로 다운로드하여 수기 대조 및 업로드.
- **To-Be:** EDI/API 직접 연동 및 자동 매칭 로직을 통한 예외 항목 중심의 관리 체계.
- **사용자 스토리:**
    - "정산 담당자로서 매일 아침 수천 건의 카드 매출과 입금 내역을 대조하는 대신, 시스템이 찾아낸 '차액 발생 건'만 집중적으로 검토하여 결산 시간을 단축하고 싶다."
- **수용 기준 (AC):**
    1. 주요 유통망(백화점, 쇼핑몰) 및 VAN사 데이터 수집 어댑터(Adapter) 구축.
    2. 판매 데이터 vs 입금 데이터 간 '자동 매칭 엔진' 가동 (금액, 승인번호 기준).
    3. 매칭 불일치 건(Tolerance 범위 외)에 대한 '미결 건 관리' 대시보드 제공.

## 2. 상품기획 (Item Planning) 핵심 요구사항

### [REQ-IM-01] 구조화된 작업지시서(PO) 버전 관리 시스템
- **As-Is:** 엑셀로 작성된 PDF/이미지 파일을 업로드하여 히스토리 추적 및 데이터 분석 불가.
- **To-Be:** 작지의 모든 항목(스펙, 도식화, 자재)을 데이터 필드화하고 버전별 변경 이력 관리.
- **사용자 스토리:**
    - "기획 MD로서 2차 샘플 제작 시 1차 대비 수정된 '소매 길이'와 '원단 변경 사항'을 시스템에서 버전 비교 기능으로 즉시 확인하고 싶다."
- **수용 기준 (AC):**
    1. 작업지시서의 주요 항목(치수표, 배색, 자재)을 DB 필드 형태로 입력 가능한 UI 제공.
    2. 변경 사항 발생 시 '새 버전 생성' 기능을 통해 이전 버전 데이터 보존(Snapshot).
    3. 버전 간 데이터 비교(Diff) 기능 및 변경된 필드 하이라이트 표시 기능.

### [REQ-IM-02] AI 기반 상품 이미지 속성 자동 분류 (Auto-Tagging)
- **As-Is:** 수십 가지의 상품 속성(실루엣, 넥라인, 기장 등)을 MD가 일일이 수기 입력함에 따른 오류 및 누락.
- **To-Be:** 상품 사진 업로드 시 AI가 형태 및 속성을 분석하여 자동으로 태깅하고 MD는 최종 컨펌만 수행.
- **사용자 스토리:**
    - "신입 MD로서 수백 개의 신상품 속성을 입력하는 시간을 줄이고, AI가 제안하는 정확한 태그를 통해 데이터 정합성을 높이고 싶다."
- **수용 기준 (AC):**
    1. 이미지 분석 API 연동을 통한 속성 추출 기능 구현 (카테고리, 핏, 소재감 등 최소 10종 이상).
    2. AI 추출 결과에 대한 '신뢰도(Confidence Score)' 표시 및 MD의 수정/확정 워크플로우 제공.
    3. 확정된 속성 데이터는 즉시 전사 마스터 데이터(Item Master)와 동기화.

## 3. 예외 케이스 및 고려 사항 (Edge Cases)
- **네트워크 장애:** EDI 연동 중 서버 장애 시 수동 업로드로 전환할 수 있는 'Fallback 모드' 필요.
- **AI 오인식:** AI가 속성을 잘못 분류했을 때 사용자가 이를 '학습 데이터 제외' 또는 '강제 수정'할 수 있는 권한 관리.
- **대량 데이터:** 수만 건의 매장 간 이동 재고 발생 시 시스템 퍼포먼스 저하 방지를 위한 비동기 처리(Queue) 도입.
