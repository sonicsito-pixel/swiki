# 영업관리팀 심층 인터뷰 결과 보고서 (PI 분석)

**작성일**: 2026-04-24
**작성자**: Gemini CLI
**대상 부서**: 영업관리팀

---

## 1. 개요
본 문서는 차세대 패션 ERP(F-ONE) 구축을 위한 영업관리팀 심층 인터뷰 내용을 바탕으로, 현행 업무의 페인 포인트(Pain Points)를 진단하고 이를 해결하기 위한 PI(Process Innovation) 과제를 정의함.

## 2. 핵심 영역별 분석 결과

### 2.1 매장 기준정보 관리 체계
- **현행 (AS-IS)**: 관리 Key가 '사업자번호'로 설정되어 점주 변경 시 데이터 연속성 단절. 시계열 분석을 위해 담당자가 수기 엑셀 작업 반복.
- **개선 (TO-BE)**: 관리 Key를 '매장코드'로 일원화. 사업자 정보는 하위 속성(SCD)으로 관리하여 점주 변경과 무관하게 매장의 전 생애주기 데이터 축적.
- **주요 과제**: 매장 마스터 구조 재설계, 과거 데이터 전환(Migration) 로직 수립.

### 2.2 매출 확정 및 정산 프로세스
- **현행 (AS-IS)**: 외부 유통망(백화점 등) 매출 데이터를 엑셀로 다운로드하여 재가공 후 업로드. 입금 내역과 판매가의 수기 대조 작업 과다.
- **개선 (TO-BE)**: EDI 및 API 연동을 통한 실시간 매출 수집. '3-Way 자동 매칭 엔진'을 통한 예외 건(차액 발생) 중심의 관리 체계 전환.
- **주요 과제**: 외부 시스템 인터페이스 구축, 자동 매칭 및 조정 전표 생성 로직 설계.

### 2.3 거점-종속 매장 수불 관리
- **현행 (AS-IS)**: 거점-종속 매장 간 재고 이동 시 가상 창고를 활용한 수기 입력 의존. 실시간 가용 재고 파악 불가 및 정산 시 매출 배분 공수 과다.
- **개선 (TO-BE)**: 시스템 내 Hub-Spoke 관계 정의. 모든 재고 이동의 트랜잭션화 및 '이동 중 재고' 위치 실시간 추적. 판매 시 자동 매출 배분 로직 구현.
- **주요 과제**: Hub-Spoke 매핑 기능, 실시간 수불 처리 엔진 고도화.

### 2.4 채권 리스크 모니터링
- **현행 (AS-IS)**: 입금 확인 및 채권 업데이트 지연으로 인해 현금 미수 사고에 대한 사후 대응만 가능.
- **개선 (TO-BE)**: 가상계좌 실시간 연동을 통한 즉시 채권 상계. 임계치 초과 미수 발생 시 즉시 알림 및 판매 제어 로직 연동.
- **주요 과제**: 뱅킹 시스템 실시간 연동, 리스크 등급별 권한 제어 로직.

### 2.5 정산 증빙(세금계산서) 자동화
- **현행 (AS-IS)**: 정산 확정 후 세금계산서 발행을 위해 데이터를 다시 가공하여 외부 시스템(Trustbill)에 입력.
- **개선 (TO-BE)**: ERP 정산 데이터 기반 Trustbill API 연동 자동 발행. 공제 항목(물류비 등)의 템플릿화로 반복 업무 제거.
- **주요 과제**: Trustbill API 연동 개발, 정산 항목-세무 항목 매핑 관리.
