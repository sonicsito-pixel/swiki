# [V7] 상품기획 MD PI 심층 인터뷰 가이드 (기준정보 및 작지 집중형)

## 1. 개요
- **목적:** 기준정보 체계 확장 및 디지털 작업지시서(W/O) 관리를 통한 상품기획 업무의 데이터 정합성 확보 및 수기 엑셀 작업 최소화.
- **핵심 타겟:** 상품기획 MD 실무진 및 팀장급.
- **제외 사항:** Centric PLM 연동 이슈 (전사 확대 전이므로 자체 관리 역량에 집중).

---

## 2. 인터뷰 핵심 질문 리스트

### [영역 1] 기준정보(Master Data) 체계 확장
> **Goal: 경직된 코드 체계를 유연화하고 분석용 데이터 속성을 강화함.**

1. **브랜드/시즌 코드 확장:**
   - 현재 브랜드 코드가 2자리로 제한되어 신규 런칭 시 어려움이 있는데, 향후 브랜드 확장성을 고려할 때 어떤 체계(영숫자 조합 등)가 적절하다고 보십니까?
   - 시즌 코드 외에 '캡슐 컬렉션'이나 '드롭(Drop)' 단위의 관리가 필요한가요?

2. **상품 속성(Attribute) 세분화:**
   - 현재 시스템에서 제공하지 않아 엑셀로 따로 관리하는 상품 속성 정보는 무엇입니까? (예: 타겟 연령층, 핏, 소재 혼용률 등)
   - 하나의 상품을 여러 카테고리로 중복 분류하여 분석해야 하는 니즈가 있습니까?

3. **마스터 데이터 정제:**
   - 미사용 컬러/사이즈 코드가 방대한데, 이를 정리하기 위한 기준(예: 3년간 미사용 시 휴면 처리 등)에 대한 의견은?

### [영역 2] 제품 정보 및 디지털 작업지시서(Work Order)
> **Goal: 엑셀 작지를 없애고 시스템 내에서 모든 이력과 데이터를 통합 관리함.**

1. **작업지시서(W/O) 프로세스:**
   - 샘플 작지에서 메인 작지로 전환될 때 정보 누락이나 재입력하는 항목은 무엇입니까?
   - 작지 확정 이후 수정 시, 이전 버전과의 변경 사항을 비교 확인하는 기능이 얼마나 절실합니까?

2. **원부자재(BOM) 연계:**
   - 제품 등록 시 원부자재 소요량(BOM) 정보를 시스템에 직접 입력하십니까, 아니면 엑셀 계산 후 최종 결과만 넣으십니까?
   - 원부자재 단가 변동이 제품 원가(Final Cost)에 실시간으로 반영되지 않아 발생하는 애로사항은?

3. **비정형 데이터 관리:**
   - 도식화(Sketch), 원단 스와치 사진, 샘플 사진 등을 시스템에 고해상도로 관리하고 생산처에 즉시 공유할 필요성이 있습니까?

### [영역 3] 입고 스케줄 및 분할 입고 (F-ONE 자체 관리)
> **Goal: 수기 엑셀 관리의 주범인 '입고 일정'의 시스템 가시성 확보.**

1. **분할 입고 유연성:**
   - 2차수 이후 입고 일정을 시스템에서 수정하지 못해 발생하는 물류/영업 부서와의 소통 오류 사례는?
   - 입고 예정일(ETD/ETA) 변경 시 영업팀에 실시간 알림이 가는 기능이 업무 효율을 얼마나 높일 수 있을까요?

- [ ] 현재 사용 중인 브랜드별 엑셀 작업지시서(W/O) 최신본
- [ ] 시즌별 상품/입고 스케줄 통합 관리 시트 (엑셀)
- [ ] 상품 코드에 담지 못해 비고란이나 별도 시트에 기록하는 상품 속성 리스트
