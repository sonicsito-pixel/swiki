# Centric PLM 도입 계획 및 FONE 연동 전략 요약

이 문서는 [원문 텍스트 자료](file:///C:/supersonic/llm_wiki/raw/sources/extracted/plm-83dc3fd156_extracted.txt)를 바탕으로 작성되었습니다. 본 자료는 2025~2027년 중장기 로드맵에 따른 Centric PLM의 단계별 도입 계획, FONE ERP 시스템과의 데이터 인터페이스 맵핑 분석, 연동 시의 기술적 제약사항, 그리고 2026년 4월 7일에 진행된 센트릭소프트(Centric Soft) 미팅 회의록을 **4단계 PI 프레임워크(As-Is, To-Be, Gap, 해결방안)**에 맞추어 종합 요약한 지식 카드입니다.

## 🗺️ PLM - FONE ERP 시스템 연계 아키텍처

```
 [ Centric PLM (Single Source of Truth) ]
   │
   ├─► 1. 기본 기획: 시즌/브랜드 트리 구조, MD 뷰 속성
   ├─► 2. 스펙/디자인: 도식화/패턴 JPEG 이미지, 치수 스펙(POM)
   └─► 3. 자재 정보: 스타일 BOM(원/부자재 소요량)
   │
   ▼ (단방향 API 연동: PLM ➡️ FONE)
 [ FONE 작업지시서 / S-ONE 소싱 ]
   - 수기 이중 입력 제거 (60여 개 컬럼 중 90% 이상 자동화)
```

## 🔍 WMS-FONE 연계 4단계 PI 진단

### 1. 작업지시서 마스터 및 자재 관리 (BOM)
* **As-Is (현행)**: 디자이너와 MD가 엑셀이나 일러스트레이터로 작업한 작업지시서와 BOM 상세 내역을 FONE ERP와 S-ONE 소싱 시스템에 각각 수기로 중복 입력함. 특히 우븐/언더웨어 등 자재 종류가 많을 경우, 수기 등록의 한계로 인해 BOM 정보를 입력하지 않고 엑셀 화면을 캡처해 이미지로 첨부하는 관행이 지속됨.
* **To-Be (목표)**: PLM 내의 스타일 및 자재 라이브러리 데이터를 FONE 작업지시서 그리드로 100% 자동 인터페이스 연계.
* **Gap (격차)**: 두 시스템 간의 단일 진실 공급원(SSOT) 결여 및 수기 중복 입력 프로세스 방치.
* **RFP 해결방안**:
  - PLM 내 '스타일 BOM'의 원/부자재 스펙 데이터(파트, 규격, 혼용률, 단가, 거래처 등)를 FONE 자재정보 그리드로 1:N API 맵핑 연동.
  - 디자이너가 Adobe Illustrator에서 작업한 도식화 및 라벨 사양 Jpeg 이미지를 PLM 체크인을 통해 FONE으로 자동 전송.

### 2. 기획 속성 및 일정/원가 연계
* **As-Is (현행)**: 브랜드, 성별코드, 소비자가, 목표원가 등 대다수의 세부 기획 및 재무 관리가 수작업 및 엑셀 메신저로 개별 공유되어 누락과 마일스톤 관리 지연이 잦음.
* **To-Be (목표)**: PLM의 트리형 시즌 계층 구조 및 MD 뷰를 활용해 속성을 시스템 제어하고 ERP에 자동 전달.
* **Gap (격차)**: 기획 마스터 데이터의 전사적 동기화 체계 미흡.
* **RFP 해결방안**: 
  - **'시즌 ➡️ 브랜드 ➡️ 부서 ➡️ 컬렉션'**으로 구성된 PLM의 계층 구조 템플릿을 FONE의 상위 코드들과 연동.
  - PLM 내 '스타일 속성(Financials 섹션)'의 목표원가 및 런칭일자를 FONE의 수주/발주 타겟 데이트와 연동.

### 3. 기술적 연동 제약 사항 및 고려점 (RFP 설계 유의)
* **공통코드 동기화**: FONE 작업지시서의 신축성, 카테고리 등은 단순 텍스트가 아닌 공통코드 기반 ComboBox 형태로 설계되어 있으므로, PLM 내 메타데이터 코드와 FONE 공통코드의 **사전 매핑 룰 정의가 필수**임.
* **데이터 타입 제한**: FONE의 DB 스펙(예: 혼용률 VARCHAR2(200), 날짜포맷 VARCHAR2(8) 등) 및 바이트 제한을 초과하는 긴 텍스트 입력 시 데이터 유실 오류 방지를 위한 글자수 제한 밸리데이션(Validation) 적용.
* **화면 고유 로직 충돌**: FONE 화면 내 '택발주확정' 버튼 트리거와 같이 화면 종속적인 유효성 체크를 우회하거나 API 통신 시 자동 트리거되도록 예외 처리.

## 📝 2026-04-07 센트릭소프트 연동 회의록 요약

1. **연동 중단 사유 및 복구 (Step 2~4)**:
   - 2025년 12월 기준 인터페이스 항목 검토 미흡으로 연동 작업이 중단된 상태를 확인.
   - **2026년 ERP 연동 검토 및 2027년 본격 연동 구축 및 확산**의 2단계 장기 로드맵 재수립.
2. **인터페이스 설계 원칙**:
   - 데이터 충돌을 원천 차단하기 위해 **양방향 연동은 지양**하고, **PLM ➡️ FONE 단방향 인터페이스** 원칙을 고수함.
3. **샘플 워크플로우 6단계 구현**:
   - 탑텐 사업부 대상 기술 표준 수립 및 품질 검증용 6단계 샘플 워크플로우(Proto ➡️ 메인보정 ➡️ QC 피팅 ➡️ FPP 공장레퍼런스 ➡️ 인라인 초도피드백 ➡️ Shipping 수납)를 PLM 내에 구현함.
4. **Approval(결재 승인) 고도화**:
   - 현재 제한적인 DRAFT/SUBMIT 기능을 넘어 BOM, SIZE, DESIGN 단위의 승인 시스템을 삼성물산 사례와 같이 사내 그룹웨어(G/W) 전자결재 API와 연동하는 추가 모듈 도입 검토.
5. **라이선스 비용 협상**:
   - 현재 14개 라이선스 사용 중 (1 User당 연간 $2,400~$2,500 ➡️ 연 5천만 원 상당).
   - 2027년 확산을 대비해 50개 라이선스 추가 구매 시 볼륨 디스카운트 단가 산정 요청 중.
