## 2. Pain Point 분석 및 근본 원인 (Root Cause)
### [현상 1] 단순 반복적 엑셀 작업 과다
- **원인**: 외부 시스템(유통망, VAN)과의 데이터 연동 표준화 부재.
- **리스크**: 담당자 부재 시 업무 마비, 결산 리드타임 증가.

### [현상 2] 데이터 정합성 오류 빈번
- **원인**: 수기 가공 과정(엑셀 편집)에서 인간의 개입으로 인한 휴먼 에러 발생.
- **리스크**: 회계 데이터 불일치 및 재정산에 따른 행정력 낭비.

### [현상 3] 예외 건 관리 체계 미비
- **원인**: 시스템 차원의 자동 대조(Matching) 기능 부재로 모든 건을 전수 조사해야 함.
- **리스크**: 대규모 할인 행사 시 업무량 폭주로 인한 분석 품질 저하.

## 3. 차세대 시스템(TO-BE) 요구사항 (RFP 반영 항목)
### [REQ-SLS-01] 외부 데이터 자동 인터페이스 엔진
- 주요 유통망(롯데/현대/신세계 등) EDI 데이터 자동 수집.
- 카드사/VAN 입금 정보 실시간 API 연동.

### [REQ-SLS-02] 3-Way Auto-Matching 엔진 구축
- ERP 매출 vs 유통망 정산액 vs 실제 입금액을 1원 단위로 자동 검증.
- 매칭 성공 건은 일괄 자동 확정 처리(Zero-Touch).

### [REQ-SLS-03] 지능형 예외 관리 대시보드
- 미매칭 건의 오차 사유 자동 분류(반품, 프로모션 차액 등).
- 화면상에서 조정 승인 시 조정 전표 자동 생성 및 회계 연동.
