## 2. 핵심 시너지 및 연결 고리 (Cross-Functional Synergy)

### ① 배분 및 입고 가시성 (상품기획 ↔ 미스토)
- **연결점:** 상품기획의 '분할 입고 일정'과 미스토의 '입고 프로세스 관제'는 상호 보완적 관계.
- **전략:** 미스토의 'WMS 실적 실시간 피드백' 기능을 상품기획 RFP의 핵심 요건으로 채택하여 MD-물류 간 소통 낭비 제거.

### ② 기준정보의 연속성 (상품기획 ↔ 영업관리)
- **연결점:** 상품기획에서 정의하는 '상품 속성(Attribute)'이 영업관리의 '다년차/다각도 매출 분석'의 기초 데이터가 됨.
- **전략:** 코드 체계를 유연화(영숫자 조합 등)하여 영업관리의 복합적인 조회 니즈를 시스템적으로 수용.

### ③ 정산 및 프로세스 자동화 (영업관리 ↔ 미스토)
- **연결점:** 영업관리의 '국내 인프라 정산' 로직과 미스토의 '글로벌 플랫폼 API 연동'은 데이터 동기화라는 동일한 목표를 가짐.
- **전략:** 배치(Batch) 위주의 정산 구조를 실시간 API 기반으로 전환하여 결산 리드타임을 획기적으로 단축.

## 3. 향후 전략적 제언 및 RFP 반영 사항

### 1) '배분 엔진(RP Engine)' 고도화
- 미스토의 사례처럼 브랜드별/유통망별로 상이한 배분 로직을 하드코딩이 아닌 **'설정(Configuration)'** 방식으로 처리할 수 있는 엔진 도입 필요.

### 2) Management by Exception (예외 관리 기반 업무)
- 정상 프로세스는 시스템 자동화에 맡기고, MD와 영업 담당자는 **'오차/지연/재고 부족 알림(Alert)'** 발생 건에 대해서만 집중하는 '워크플로우 알림 체계' 구축.

### 3) 전사 공통 데이터 거버넌스 수립
- 상품기획(생성) → 영업관리(매출) → 물류(출고)로 이어지는 데이터 생애주기를 관리하기 위해, 시스템 설계 단계부터 **'마스터 데이터 관리(MDM)'** 관점의 정합성 확보.
