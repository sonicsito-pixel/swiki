# [차세대 패션 ERP] 상품기획팀 PI 심화 인터뷰 가이드 (V4)

**근거자료**: FA-ONE 프로세스 진단 보고서 및 신규 업무 니즈 (2026.04)
**인터뷰 목적**: 상품정보등록 일관성 확보, 샘플 관리 프로세스 신설, FONE-SONE 정보 연계 강화

---

## 1. 상품 기준정보(Master Data) 거버넌스 및 확장성
*   **[이슈] 브랜드 코드 체계 확장성**: 현재 1자리 알파벳 체계의 잔여 코드 부족으로 신규 브랜드 런칭 시 한계 예상.
    - **질문**: 브랜드 코드를 2자리 또는 영숫자 조합으로 변경할 경우, 스타일 관리나 과거 데이터 분석 측면에서 우려되는 사항이 있습니까?
*   **[이슈] 칼라/사이즈 코드 표준화**: 미사용 코드 과다 및 전사/브랜드별 사이즈 체계 이원화.
    - **질문**: 데이터 정합성을 위해 사이즈 체계를 표준화하거나 브랜드별 매핑 테이블로 통합 관리하는 것에 대한 현업의 수용도는 어떠합니까?

## 2. [신규] 샘플 관리(Sample Management) 프로세스 니즈
*   **[현황] 시스템 기능 부재**: 현재 샘플의 요청, 진행, 피드백 과정이 시스템화되어 있지 않아 수기나 별도 툴로 관리 중임.
    - **질문 1**: 개발 샘플(1차, 2차 등) 요청 시 협력업체와의 커뮤니케이션과 진행 현황(수령 여부 등)을 현재 어떤 방식으로 기록/추적하고 계십니까?
    - **질문 2**: 샘플 수령 후 수정 사항(피드백)이 실제 작업지시서(Tech Pack)에 반영되는 과정에서 정보 누락이나 중복 입력이 발생하는 지점은 어디입니까?
    - **질문 3**: 실물 샘플의 위치(보관 장소, 대여 현황 등) 관리 및 폐기 이력 관리가 시스템적으로 필요하다고 느끼십니까?

## 3. FONE-SONE 정보 연계 및 작지 관리
*   **[이슈] 패션관리(FONE)-소싱(SONE) 연계 강화**: 상품 정보의 일관성 있는 전송 및 동기화.
    - **질문**: 기획 품번 정보가 소싱 시스템의 Tech Pack으로 전환될 때, MD가 수기로 재입력해야 하는 항목 중 자동화가 가장 시급한 데이터는 무엇입니까?
*   **[이슈] 원가 정보의 환류**: 소싱 시스템의 확정원가 데이터를 기획 화면에서 실시간 참조.
    - **질문**: 소싱본부에서 확정된 최종 원가가 실시간으로 공유될 경우, MD의 시즌별 수익성 예측 및 리오더 의사결정에 어떤 도움이 되겠습니까?
*   **[이슈] TAG 발주 프로세스 효율화**: TAG 발주 수기 작업의 시스템 연동 검토.
    - **질문**: 작지 확정과 동시에 TAG 발주 데이터가 자동 생성된다면, MD가 시스템에서 최종 컨펌해야 할 핵심 변수(가격, 케어라벨 등)는 무엇입니까?

## 4. 스타일 속성 분석 및 조회
*   **[이슈] 스타일 속성 기반 통합 조회**: 다년차/시즌 통합 및 속성(소재, 실루엣 등)별 상세 분석 요구.
    - **질문**: 과거 데이터를 검색/분석할 때 현행 시스템의 필터링 기능에서 가장 개선이 필요한 부분은 무엇입니까? (예: 특정 소재를 사용한 모든 시즌 스타일 일괄 조회 등)

---

### [현업 확인 사항 요약]
1.  **샘플 관리**의 시스템화 범위 (요청/현황/피드백/이력).
2.  **브랜드 코드 확장**에 따른 기존 데이터 매칭 로직.
3.  **FONE-SONE 연계**를 통한 중복 입력(Double-work) 제거 항목 도출.
