1. 영업관리 프로세스 맵 전체 구조 (Overview)
-------------------------------------------------------------------------------

[P1. 기준정보]
- 주요 업무: 매장 등록, 거점/종속 관계 설정, 계약 조건 관리
- 현행 문제(As-Is): 사업자 변경 시 데이터 단절, 부서 간 입력 데이터 간섭
- 개선 방안(To-Be): SCD 기반 고유 매장ID 체계 구축, 부서별 전용 입력 탭(Tab) 분리

[P2. 공급분 매입]
- 주요 업무: 공급업체 발주, 매입 확정 처리, 매입 단가 관리
- 현행 문제(As-Is): 수기 입력 과다, 불필요한 가상 트랜잭션으로 인한 복잡도
- 개선 방안(To-Be): 물류 연동 매입 자동 확정 로직, 관리 단계 축소 및 슬림화

[P3. 매출 및 정산]
- 주요 업무: 유통망 매출 수집, 유통망-ERP 대조, 중간관리자 정산 계산
- 현행 문제(As-Is): 수기 엑셀 기반 VLOOKUP 대조 작업, 정산 공제 항목 수기 계산
- 개선 방안(To-Be): 3-Way(ERP-유통망-입금) 자동 매칭 엔진, 정산 항목 템플릿화

[P4. 재고 및 수불]
- 주요 업무: 매장 간 이동(RT), 자가소모 처리, 물류 반품 관리
- 현행 문제(As-Is): 이동 중 재고(In-transit) 추적 불가, 자가소모 수기 전표 처리
- 개선 방안(To-Be): 이동 중 재고 실시간 가시성 확보, 현장 모바일 자가소모 등록

[P5. 리스크 및 채권]
- 주요 업무: 입금 확인, 채권 잔액 상계, 리스크 기반 판매 제어
- 현행 문제(As-Is): 실시간 채권 파악 지연, 미수 매장 판매 제어 수기 처리
- 개선 방안(To-Be): 실시간 CMS 채권 상계, 시스템 자동 판매 제어(Blocking)

-------------------------------------------------------------------------------
2. 상세 업무 Flow 및 Pain Point 매핑
-------------------------------------------------------------------------------

[P1] 매장 기준정보 관리
- 업무흐름: 신규 매장 확보 -> 사업자 등록 -> 매장 코드 생성 -> 계약 정보 입력
- 고통지점(PP): 사업자 변경 시 과거 매출 이력과 단절되어 데이터 통계 왜곡 발생.
- 개선모델: 고유 ID 자동 채번 및 SCD 엔진을 통한 시점별 속성 변경 이력 관리.

[P2] 공급분 매입 프로세스
- 업무흐름: 발주서 생성 -> 물류 입고 -> 매입 전표 수기 생성 -> 거래처 확정
- 고통지점(PP): 물류 입고 데이터가 있음에도 영업 담당자가 매입 전표를 중복 생성.
- 개선모델: 입고 시 매입 자동 확정(Auto-Matching) 및 가상 트랜잭션 단계 축소.

[P3] 매출 및 정산 프로세스
- 업무흐름: 일 매출 발생 -> 유통망 EDI 수집 -> 카드 입금 대조 -> 정산/공제 계산
- 고통지점(PP): 수천 개 매장의 매출 데이터를 엑셀로 대조하며, 공제 항목 누락 빈번.
- 개선모델: EDI 자동 수집 및 3-Way 자동 매칭 엔진, 정산서 역발행 자동화.

[P4] 재고 수불 프로세스
- 업무흐름: 거점 매장 입고 -> 종속 매장 재고 이동 -> 판매 -> 매장 간 정산
- 고통지점(PP): 이동 중인 재고의 위치 추적 불가 및 YA센터 등 가상창고 오남용.
- 개선모델: RT 발송 시 이동 재고 가시화, 가상 창고 폐쇄 및 물리 위치 기반 수불.

[P5] 채권 및 리스크 관리
- 업무흐름: CMS 입금 수집 -> 채권 상계 -> 미수금 확인 -> 판매 제어
- 고통지점(PP): 채권 파악 지연으로 미수 매장에 계속 출고되는 리스크 발생.
- 개선모델: 실시간 채권 상계 및 임계치 초과 시 시스템 자동 판매 Blocking.
