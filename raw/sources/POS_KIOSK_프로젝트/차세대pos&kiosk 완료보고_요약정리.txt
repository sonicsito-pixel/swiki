### `SSTS-STM-05-완료보고서(V1.0)-F.pdf` 분석 결과 (한글 번역)

이 문서는 CJ올리브네트웍스에서 수행한 "신성통상 차세대 POS 및 키오스크 구축 프로젝트"의 완료 보고서입니다. 이 프로젝트는 기존 WEB POS를 CS POS 및 키오스크 시스템으로 전환하고, AWS 클라우드 기반의 POS 관리 및 API 시스템을 구축하는 것을 목표로 했습니다. 신성통상의 6개 브랜드에 걸쳐 1,270개 매장에 시스템을 체계적으로 확산하여 안정적인 운영을 보장하는 것이 목표였습니다.

**1. 주요 내용 요약:**

**프로젝트 개요 (I. 프로젝트 개요):**
이 프로젝트는 총 12개월('25년 2월 ~ '26년 2월) 동안 진행되었으며, 시스템 무결성 확보를 위해 11회의 통합 테스트와 2회의 성능 테스트를 거쳤습니다. 전 매장으로의 단계적 확산을 위해 3회의 오픈 워크샵과 9회의 사용자 교육을 실시했습니다.

**구축 내용 (II. 구축 내용):**
POS/키오스크와 F-ONE 영업 관리 시스템 간의 인터페이스를 위한 API를 모듈화하여 안정성과 연결성을 강화했습니다.
*   **POS 시스템 설치:** POS 및 키오스크 프로그램 설치 파일은 중앙에서 관리하여 접근성을 높였으며, OS 환경, 프레임워크, 데이터베이스, 장치 드라이버별 설치 시 사용자 편의성을 위해 별도의 설정 없이 자동 설치되도록 패키지화했습니다.
*   **POS UI/UX:** 정보의 중요도에 따라 4-Layer 표준 프레임워크를 적용하고, 복잡한 POS 기능을 5개 영역으로 단순화하여 직관적인 업무 처리와 매장 운영 효율성을 극대화했습니다.
*   **POS 주요 기능:** 다양한 결제 수단(현금, 카드, 간편 결제), 자동 프로모션 적용, 영수증 조회, 매출 조회, 장비 제어, 기준 정보 동기화 등 판매 및 관리를 위한 관리자 화면 기능을 제공합니다.
*   **키오스크 주요 기능:** 고대비, 확대, 음성 안내, 눈높이 조절 등 배리어프리 표준을 적용하여 고객의 접근성을 보장하고, 쿠폰/멤버십 인증, 관리자 기능 등 다양한 편의 기능을 제공합니다.
*   **POS 관리 시스템:** 3-Layer 표준 프레임워크를 적용하여 최적의 UI/UX를 구현했습니다. 메시지 관리, 시스템 환경 설정, 제품 관리, 매장 관리, POS/키오스크 장비 환경 관리 등의 기능을 포함하며, 긴급 배포 시 마스터 데이터를 전체 또는 브랜드별로 업데이트할 수 있습니다.

**주요 성과 (III. 주요 성과):**
*   **아키텍처 개선:** AWS 클라우드 아키텍처로 전환하여 자원 확장이 가능한 고성능/고가용성 인프라를 구축했습니다. 클라우드 환경을 중심으로 POS 및 영업 관리 시스템 간 성능을 확보했으며, 에이션패션을 위한 인프라 서비스는 중지되었으나 언제든 서비스 가동을 위한 환경은 구축되어 있습니다.
*   **안정성 확보:** 무중단 결제 아키텍처를 구현하여 네트워크 장애 등 비상 상황에서도 현장 판매가 지속될 수 있는 환경을 구축했습니다.
*   **다양한 결제 수단 제공:** Apple Pay, Tax Refund와 같이 장비 특성 및 정책으로 제약이 있는 결제 수단을 제외하고 모든 결제 수단을 추가하여 고객의 차별화된 구매 경험과 매장 수익성 향상에 기여했습니다.

**제언 (IV. 제언):**
차세대 영업 및 매장 시스템과 키오스크 고도화에 대한 제언을 통해 비즈니스 변화와 요구사항에 즉각적으로 대응하고 시스템 성능 개선, 데이터 통합, 운영 확장성 확보, 글로벌 시스템으로의 기반 마련을 강조합니다.

# 차세대 POS & KIOSK 기술스텍

기간계시스템(패션시스템)과 차세대 POS 시스템의 데이터 연동 기술 스택:

전반적으로, 기존 기간계시스템(주로 On-Premise 환경)과 새로 구축된 차세대 POS 시스템(AWS Cloud 환경) 간의 연동은 API 중심의 마이크로서비스 아키텍처와 클라우드 네이티브 기술을 활용하여 이루어지고 있습니다.

1. 클라우드 플랫폼:
- AWS (Amazon Web Services): 차세대 POS 시스템의 모든 인프라는 AWS 클라우드 위에 구축되었습니다. 이는 확장성, 안정성, 고가용성을 확보하는 기반이 됩니다.
1. API 및 연동 기술:
- API Gateway (AWS API G/W): 외부 시스템과의 연동 및 내부 서비스 간 통신을 위한 API의 진입점 역할을 합니다. 특히 "영업관리 API" 및 "POS관리 API"는 이 게이트웨이를 통해 노출되거나 내부적으로 사용될 수 있습니다.
- VPC Link: API Gateway와 AWS Virtual Private Cloud(VPC) 내의 리소스(예: NLB/ALB)를 안전하게 연결하는 데 사용됩니다.
- 영업관리 API / POS관리 API: 이 커스텀 API들은 기간계시스템과 차세대 POS 시스템 간의 데이터 교환을 담당하는 핵심적인 비즈니스 로직을 포함합니다.
    - 기준정보 송/수신: 기간계시스템(예: F-ONE)에서 필요한 마스터 데이터(매장, 브랜드, 상품, 프로모션, 가격 정보 등)를 차세대 POS 시스템으로 전송합니다.
    - 판매정보 송/수신: 차세대 POS 시스템에서 발생한 판매 데이터를 기간계시스템으로 다시 전송합니다.
- Batch Processing: "POS관리 Batch"라는 언급을 통해, 대량 데이터 처리나 스케줄링된 동기화 작업(예: 일일 판매 집계, 재고 동기화)에 배치 처리가 활용됨을 알 수 있습니다.
1. 데이터베이스:
- Amazon Aurora PostgreSQL: 차세대 POS 시스템의 주력 데이터베이스로, 관리형 서비스로서 높은 성능과 확장성을 제공합니다.
- Oracle DB (On-Premise): 기존 기간계시스템은 On-Premise 환경의 Oracle DB를 사용하고 있는 것으로 추정됩니다. 차세대 POS 시스템은 이 기존 DB와의 연동 또는 데이터 마이그레이션을 통해 운영됩니다.
1. 컴퓨팅 및 컨테이너 오케스트레이션:
- AWS ECS (Elastic Container Service): POS, POS 백엔드, POS관리시스템, API 서비스 등 차세대 POS 시스템의 애플리케이션은 ECS 컨테이너 위에서 실행됩니다. 이는 마이크로서비스 아키텍처를 지원하고 배포 및 관리를
용이하게 합니다.
- ALB (Application Load Balancer), NLB (Network Load Balancer): 트래픽을 ECS 서비스로 분산하여 시스템의 가용성과 성능을 보장합니다.
1. 네트워킹 및 보안:
- AWS VPC (Virtual Private Cloud): 클라우드 내에서 격리된 네트워크 환경을 구축하여 보안성을 확보합니다. COM VPC (공통), POS/키오스크 관리 VPC, 온라인몰 API VPC 등으로 세분화되어 있습니다.
- Site-to-Site VPN: On-Premise 환경의 기간계시스템과 AWS 클라우드 간에 안전하고 암호화된 네트워크 연결을 제공하여 내부 데이터 연동의 보안을 강화합니다.
- AWS WAF (Web Application Firewall), CloudFront, Shield: 외부로부터의 웹 공격 및 DDoS 공격을 방어하여 시스템의 안정성을 확보합니다.
- IAM (Identity and Access Management): AWS 리소스에 대한 접근 권한을 관리하여 보안 거버넌스를 유지합니다.
1. 모니터링 및 로깅, 개발 도구:
- CloudWatch, OpenSearch, DATADOG, JENNIFER: 시스템 성능 모니터링, 로그 분석, APM(Application Performance Management)을 통해 시스템 상태를 실시간으로 감시하고 문제 발생 시 신속하게 대응합니다.
- SonarQube, GitLab: 개발 프로세스 내에서 코드 품질 관리 및 버전 관리에 사용되는 도구입니다.

이러한 기술 스택을 통해 신성통상의 차세대 POS 시스템은 기존 기간계시스템과의 안정적인 데이터 연동을 기반으로 현대적이고 확장 가능한 인프라를 구축했습니다.
