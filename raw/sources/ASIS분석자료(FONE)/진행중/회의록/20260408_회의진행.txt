

20260408 회의 준비

목적 : 차세대 패션 관리 시스템 구축 관련하여
기존 시스템의 프로세스에 대해 검토하고 기존 시스템에서 이슈가 될 만한 부분에 대해 개선점에 대한 협의

1. 기존 패션관리시스템 업무 별 프로세스 확인

2. 업무 별 이슈 사항 제기

- 회의에서는 AGENDA 논의 ( 이슈 제기 )

	1. 기준정보(Master Data) 관리 진단
	
	브랜드: 상위+서브 코드 조합 → 매출/물류/BI 분석 기준
	품종: 상위/하위 구조, 회계는 상위 품종만 매핑 → 효율성 확보
	칼라/사이즈 관리
	칼라: 1,434개 중 실제 650개만 사용 → 불필요 코드 정리 필요
	사이즈: 전사 표준 + 브랜드별 기획 사이즈 이원화 구조
	핵심 이슈 및 개선 방향
	브랜드 코드 1자리 체계 → 확장 한계 임박 → 2자리/영숫자 체계 전환 필요
	사이즈 체계 → 표준화/통합 필요

	-> 결론: 기준정보 거버넌스 정비가 전체 데이터 품질의 핵심

	2. 상품기획 및 PLM 연동 전략
	
	작업지시서 : PLM 연동 (Centric) 을 통해 데이터 일관성 확보	
	정보 일관성 유지: 제품 속성, 작업지시, 스타일 정보 간의 일관성 유지 및 변화 관리가 필요하며, 이를 해결하기 위해 PLM(Centric) 시스템으로의 대체를 검토 중입니다

	-> 결론: PLM 중심으로 기획–소싱–물류 데이터를 연결하여 디지털 전환 가속

	4. 매장 정보 관리
	수기 관리에 따른 오입력 위험: 매장 ID 등이 대개 규칙성 있게 부여되지만, 수기로 입력되고 있어 오입력 가능성이 존재합니다
	히스토리 관리 미흡 : 매장 관련 데이터의 변경 관리가 부족하여 과거 이력을 파악하기 어렵습니다
	기능 및 분류 정리: 사용하지 않는 기능을 제거하고, 데이터 분석을 고려하여 다양한 매장 유형 및 분류를 정리할 필요가 있습니다

	5. 상품 기획
	TAG 발주 수기 작업: 브랜드 특성에 따라 수기로 처리하는 TAG 발주 업무를 자동화(RFID 도입 연계 등)할 수 있는지 확인이 필요합니다

	6. 물류 관리 (입/출고 및 재고)
	창고 기준 불일치: FA-ONE과 물류 시스템(WMS) 간의 창고 구분이 상이하여 수불 처리가 모호하므로, 창고 일원화 관리가 요구됩니다
	온라인/이관 수불 실적 반영:	온라인 창고를 매장으로 운용하고 있어, 입출고 수불 시 매장과 창고 간 불필요한 이동 이력이 생성되는 이슈가 있습니다.
	BLIND 반품 처리 프로세스가 부재(2026년 7월 개선 예정)하며, 정상/불량 재고 이관 시 입출고 수불 실적을 어떻게 반영할지 세부 CASE 확인이 필요합니다
	자가소모 처리 프로세스 한계: 매장이나 온라인에서 자가소모 처리를 할 때, 전산상 반드시 물류 센터(창고)로 재고를 선 이동(매장 반품)시킨 후 진행해야 하는 구조적 제약이 있습니다

	7. 영업 기획 (매출 목표)
	목표 히스토리 관리 부재: 매장 매출 목표 변경에 대한 히스토리 관리가 되지 않아, 목표 변경의 근거 파악과 목표 대비 실적 데이터의 신뢰성 확보에 어려움이 있습니다

	8. 매장 업무 (점간 이동)
	이동 중 재고 처리 모호성: 매장 간 점간 이동(RT) 시, 인수 매장이 확정하기 전까지 해당 재고가 인도 매장의 재고로 남아 있는 이슈가 있습니다
	이에 따라 가상 창고에 재고 수불을 반영할 필요성이 있는지 검토가 필요합니다 ( 현재는 확정 시 동시 진행 )

	9. 영업 관리 (정산 및 기타 업무)
	정산 수기 업무 전산화 및 자동화:
	유통망별 EDI 정보 수집 기간을 단축하기 위해 RPA(로봇 프로세스 자동화) 적용이 가능한지 파악이 필요합니다
	중간관리자 정산 시 수기로 관리 중인 각종 공제 내역(판/부자재, 수선비 등)의 전산화 가능성 검토가 필요합니다 ( 수기 관리 )

- 회의에서 다룰 AGENDA 를 검토하고, AGENDA 에 대한 솔루션을 도출 해 내는 목적

- 키 이슈를 그대로 두고 작은 이슈들에 대해 파악 해 볼 것

- 전체 프로세스를 그리고 단계 별로 세부 프로세스들을 채워 넣어 프로세스 완성도를 높이자

- 업무 영역 및 기능에 대한 세부적인 분석 필요

- 패션 관리 시스템(FONE) 과 인터페이스 하고 있는 시스템들의 영향도 파악

- 패션 관리 시스템의 데이터 및 프로세스가 변경 될 때 연관 시스템에 영향을 주지 않는지 확인 필요


- 이커머스, WMS, POS, BI 데이터 등

- 필요한 화면 - 기능에 대한 파악 ( CORE PROCESS )

- 비슷하거나 관련 있는 화면 산재 -> 통합 화면 구성 검토

- 불필요한 분석 조회 화면 -> BI 로 전환 고려

- 차세대 구축 시 데이터 마이그레이션 범위에 대한 고려 필요

- 몇년 데이터까지 차세대 데이터로 가져갈지 고려

- 기존 데이터에 대한 아카이빙 방안

- 추가 검토 사항

- RFID 도입 ( 3년 내 도입 가능성 ) : 도입 범위 및 비용, 기존 프로세스에 어떤 영향을 미칠지 파악

- DX 최적화 준비 ( 데이터, AI, 자동화... )
