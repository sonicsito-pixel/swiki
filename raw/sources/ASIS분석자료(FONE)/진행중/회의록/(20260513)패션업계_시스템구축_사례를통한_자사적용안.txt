 1. 주요 패션업계 차세대 PI 구축 사례

 ① Descente Korea

* 부서별 AS-IS 상세 분석보다 ‘마스터급 간담회’ 중심으로 개선 방향 도출
* ERP 재구축 → 분석/협업 고도화 → AI 분석 단계로 이어지는 5개년 장기 로드맵 추진
* AI 활용을 고려하여 초기 단계부터 데이터 축적 체계 설계
* 13~14년치 데이터 전체 이관 수행
* 성능 저하 요인이던 MView 기반 집계 구조 제거로 시스템 경량화 진행
* 클라우드는 OCI 기반으로 전환하여 비용 절감 및 Oracle GoldenGate 활용

 ② Amer Sports Korea

* 글로벌 SAP 시스템과 국내 운영 프로세스 간 Gap 분석 중심 PI 수행
* 수입 기반 사업 특성에 맞춰 통관·부대비용 포함 원가관리 체계 강화
* 내부통제 및 데이터 정합성 확보를 핵심 과제로 설정
* 시스템 오픈 이후 Change Management 중요성 강조

 ③ K2 Korea

* 브랜드 및 사업 카테고리 확장 대응을 위한 통합 업무 플랫폼 구축
* 단순 조회형이 아닌 현장 영업 실무 수행 가능한 모바일 업무 플랫폼 제공
* 약 20여 개 업무 유형을 모바일 기반으로 처리 가능하도록 프레임워크화

 ④ Shinsegae International / ABC-MART

* 여성복 세트 상품(투피스·쓰리피스) 특성을 고려한 세트 단위 운영 로직 적용
* 배분·반품·가격·로테이션 등을 단품이 아닌 세트 기준으로 관리
* CRM과 POS 연계를 통한 실시간 프로모션 및 옴니채널 기능 강화

 2. 차세대 시스템 기술 및 인프라 전략 주요 시사점

 ① 클라우드 및 DB 전략

* Oracle → PostgreSQL 등 오픈소스 전환은 기존 함수·프로시저 재개발 부담이 매우 큼
* 실무적으로는 Oracle 유지 전략이 현실적이라는 사례 다수 존재
* OCI 선택 이유

  * AWS 대비 비용 절감
  * Oracle GoldenGate 무상 지원
  * Oracle DB 친화적 운영 환경
* 이미지·도면 등 주요 회사 자산 이며, 대용량인 파일은 별도 스토리지 분리 저장 방식 활용 필요

 ② API 기반 인터페이스 표준화

* 기존 EAI 방식은 비용·유지보수·유연성 측면에서 한계 존재
* 최근 트렌드는 RESTful API 기반 API Gateway 구조 채택
* ERP와 외부 시스템 간 인터페이스를 단일 창구로 통합
* 실시간 데이터 연계 및 정합성 확보가 핵심 방향

 ③ 기준정보(Master Data) 및 코드 체계 개편

* 기준정보 개편 전 전사 영향도 분석 필수
* 신코드/구코드 병행 운영 전략 필요
* 기존 재고 및 운영 데이터는 연차별 자연 소멸 방식으로 관리
* 중앙 MDM 체계를 통해 기준정보 거버넌스 확보 필요

---

 3. 신성통상 차세대 PI 추진 시 핵심 시사점

 전략적 관점

* 빅뱅 방식보다 단계별 고도화 전략이 현실적
* 초기부터 AI 활용을 고려한 데이터 구조 설계 필요
* Change Management 및 내부통제 체계 확보 중요

 시스템 관점

* Oracle 유지 + 클라우드 전환 조합이 현실적 대안 가능성 높음
* API Gateway 중심 인터페이스 구조 검토 필요
* 대용량 파일 분리 저장 아키텍처 고려 필요

 운영 관점

* 기준정보 개편은 단순 코드 변경이 아닌 전사 업무체계 재정의 수준 접근 필요
* MDM 중심 기준정보 통제 체계 구축 필요
* 모바일 기반 업무 플랫폼 확대 가능성 검토 필요

---

 4. 종합 결론

이번 벤치마킹 사례들은 단순 ERP 교체가 아니라,
“데이터 구조 표준화 + API 중심 연계 체계 + 클라우드 기반 확장성 + 기준정보 거버넌스”를 핵심 축으로 차세대 시스템을 설계하고 있음을 보여줌.

특히 패션업계 특성상:

* 상품/세트 운영 구조
* 시즌성 재고
* 멀티 브랜드 운영
* 온·오프라인 옴니채널
* 글로벌 물류 및 원가 관리

등을 고려한 구조 설계가 중요하며,
신성통상 또한 단기 구축보다 단계적 PI 및 데이터 중심 아키텍처 전략 수립이 필요하다는 시사점을 도출함.
