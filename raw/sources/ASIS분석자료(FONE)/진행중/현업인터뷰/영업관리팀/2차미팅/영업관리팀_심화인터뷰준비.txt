[영업관리 PI 심화 분석 보고서 - 업무 단위별 정리]

작성일: 2026-04-23
작성자: Gemini CLI

================================================================================
1. 매장 기준정보 체계 혁신
================================================================================
* 결과(산출물, 업무 단위)
- 통합 매장 마스터 데이터 / 매장별 이력 관리 체계

* 작업과정(시스템 또는 수기작업, 실제로 진행되는 업무 과정) 
- 1. 신규 매장 오픈/변경 결정 2. 사업자번호 확인 3. 과거 사업자번호별 데이터 검색(수기) 4. 엑셀을 통한 매출/재고 이력 매핑 5. ERP 등록

* 왜? ( 업무 개선 요인-pain point, 현업 요청 개선 내용 )
- 왜 매장 정보를 엑셀로 따로 관리해야 하는지? ( 시스템의 문제인지, 기존에 진행하던 관례적 업무인지 파악 )

* pain point 분석
- 매장 관리의 Key가 '사업자번호'로 되어 있어 사업장 변경 시 데이터 연속성이 단절됨. 이로 인해 동일 위치 매장의 시계열 분석을 위해 담당자가 수기 엑셀 작업을 반복함.

* 개선 방향
- 관리 Key를 '매장코드'로 일원화하고 사업자정보를 하위 속성(SCD)으로 관리. 점주가 바뀌어도 매장코드는 유지되어 별도 작업 없이도 매장의 전 생애주기 데이터가 자동 축적되도록 함.

2. 매출 확정 및 정산 자동화 (Zero-Manual)
================================================================================
* 결과(산출물, 업무 단위)
- 일일 매출 확정 전표 / 유통망 정산 리포트

* 작업과정(시스템 또는 수기작업, 실제로 진행되는 업무 과정) 
- 1. 유통망 매출 엑셀 다운로드 2. ERP 양식에 맞춰 재가공 3. 시스템 업로드 4. VAN사 입금 내역 대조(수기) 5. 차액 발생 시 원인 추적 6. 최종 확정

* 왜? ( 업무 개선 요인-pain point, 현업 요청 개선 내용 )
- 백화점 매출과 카드사 입금 내역을 왜 사람이 일일이 대조하고 있는지? ( 시스템적인 문제? 현재 시스템에서 해결이 불가능한 원인 파악 )

* pain point 분석
- 외부 데이터(EDI, VAN) 연동 부재로 인해 단순 반복성 엑셀 작업에 과도한 공수가 투입됨. 이로 인해 결산이 늦어지고 데이터 오입력 리스크가 상존함.

* 개선 방향
- EDI 및 API 직접 연동을 통해 수기 업로드 공정 제거. '자동 매칭 엔진'을 도입하여 정합성이 맞는 건은 자동 확정하고, 차액 발생 건만 팝업으로 관리자에게 알려 처리하는 '예외 관리' 방식으로 전환.

3. 중간관리자 및 거점매장 수불 , 매출 배분 시스템화
================================================================================
* 결과(산출물, 업무 단위)
- 거점별 손익 리포트 / 중간관리자 수수료 정산서

* 작업과정(시스템 또는 수기작업, 실제로 진행되는 업무 과정) 
- 1. 매장 간 재고 이동 발생 2. 유선/채팅으로 상호 확인 3. 가상 창고(X, OA)를 활용한 수기 입력 4. 월말 매출 배분 로직 엑셀 계산 5. 정산 반영

* 왜? ( 업무 개선 요인-pain point, 현업 요청 개선 내용 )
- 거점매장과 종속매장 사이의 재고가 왜 실시간으로 파악되지 않고 가상 창고에 묶여 있는지?

* pain point 분석
- 거점-종속 관계가 시스템상에 정의되어 있지 않아 수불 히스토리가 불투명함. 재고 로스 발생 시 책임 소재 파악이 어렵고 정산 시 매출 재배분에 많은 시간이 소요됨.

* 개선 방향
- 기준정보에 거점(Hub)-종속(Spoke) 구조를 등록. 매장 간 모든 이동을 시스템 트랜잭션으로 관리하여 '이동 중 재고' 가시성 확보. 종속 매장 매출 발생 시 거점 매장으로 자동 합산/배분되는 정산 로직 구현.

4. 채권 및 미수금 실시간 리스크 관리
================================================================================
* 결과(산출물, 업무 단위)
- 실시간 미수금 현황판 / 사고 예방 알림

* 작업과정(시스템 또는 수기작업, 실제로 진행되는 업무 과정) 
- 1. 은행 입금 확인 2. 엑셀 미수 대장 업데이트 3. ERP 업로드 4. 보고서 생성 5. 관리자 사후 검토 6. 미수 과다 매장 유선 독촉

* 왜? ( 업무 개선 요인-pain point, 현업 요청 개선 내용 )
- 100만 원 이상 현금 미수가 발생했는데 왜 며칠 뒤에야 보고서를 보고 알게 되는지? ( 실시간 입금 관리 불가 원인 파악 )

* pain point 분석
- 입금 데이터 반영이 실시간이 아니며, 리스크 기준(예: 미수 100만 원 초과)에 따른 자동 경고 체계가 없음. 직영점 등 현금 사고 위험에 대한 선제적 대응이 불가능함.

* 개선 방향
- 가상계좌 실시간 연동으로 즉시 채권 상계 처리. 특정 금액 이상 미수 발생 시 담당자/팀장 시스템 로그인 시 '즉시 알림 팝업'을 노출하여 즉각적인 조치가 가능하도록 설계.

================================================================================
5. 중간관리자 수수료 및 증빙 자동화
================================================================================
* 결과(산출물, 업무 단위)
- 중간관리자 정산 세금계산서 발행 / 비용 지급

* 작업과정(시스템 또는 수기작업, 실제로 진행되는 업무 과정) 
- 1. 중간관리자 등록 2. 수수료율 등록 3. 정산 데이터 생성 4. 공제내역 수기 작업 5. 시스템 업로드 6. 트러스빌 세팅 7. 발행 8. 지급

* 왜? ( 업무 개선 요인-pain point, 현업 요청 개선 내용 )
- 정산이 다 끝났는데 왜 세금계산서 발행을 위해 또 수기로 데이터를 만져야 하는지? ( 정산 / 실 세금계산서 간 금액 차이 발생원인, 신뢰성 결여에 대한 원인 파악 )

* pain point 분석
- ERP 정산 결과와 외부 세금계산서 발행 시스템(Trustbill) 간 연동이 미흡함. 특히 공제 항목(물류비, 판촉비 등)이 수기로 관리되어 발행 단계에서 데이터 중복 입력 발생.

* 개선 방향
- ERP 정산 확정과 동시에 Trustbill API를 호출하여 세금계산서(역발행) 자동 생성. 복사 및 일괄 적용 기능을 강화하여 매월 반복되는 공제 항목 설정을 자동화함.
