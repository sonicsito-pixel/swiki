[신성통상 차세대 FA-ONE 구축] 현업 인터뷰 및 심층 분석 가이드 (마스터 정보 & 영업관리)

1. 마스터 정보 (Master Data) 영역

마스터 정보는 시스템의 '데이터 표준'을 세우는 기초 공사입니다. 확장성과 무결성 확보에 집중합니다.

[접근 방식: 현업 인터뷰 & 분석]
* 브랜드 코드 확장성
  - 질문: "신규 브랜드 론칭 시 코드 부여의 어려움은? 현재 1자리 체계가 향후 5년 내 브랜드 확장 계획을 수용 가능한가?"
  - 분석: 현재 A~Z 코드 사용 현황 및 폐기 브랜드 코드 재사용성 파악. 2자리 확장 시 POS, WMS, 회계 등 연계 시스템 인터페이스 영향도 분석.
* 칼라/사이즈 표준화
  - 질문: "미사용 칼라 코드가 대량 발생한 근본 원인은? 브랜드별 사이즈 체계 차이로 인한 기획/생산 오류 사례는?"
  - 분석: 중복/유사 칼라 매핑 테이블 작성. 브랜드별 사이즈 그룹화(Grouping) 규칙 도출.
* 매장 마스터 관리
  - 질문: "매장 ID 수기 입력 시 발생하는 오입력 사례는? 유통망별(백화점/대리점 등) 관리 필수 속성은 무엇인가?"
  - 분석: 매장 ID 자동 채번 로직 설계. 정보 변경(점주 교체 등) 이력 관리를 위한 데이터 스펙 검토.

[도출해야 할 결론 (To-Be)]
- 브랜드 코드 2자리 확장으로 향후 10년 이상의 비즈니스 확장성 보장.
- 미사용 코드 정리 및 신규 코드 생성 승인 프로세스(Governance) 수립.
- 매장 ID 자동 생성 및 정보 변경 이력(History) 자동 기록 체계 구축.

2. 영업관리 (Sales Management) 영역 (※ 영업기획 업무 제외)

영업관리는 '수기 업무의 전산화'와 '현장 운영의 효율성' 확보가 핵심입니다.

[접근 방식: 현업 인터뷰 & 분석]
* 정산 및 공제 업무 (Pain Point)
  - 질문: "엑셀로 따로 관리하는 판/부자재비, 수선비 등 수기 정산 항목 리스트는? EDI 데이터 수집/검증 소요 시간은?"
  - 분석: 수기 관리 엑셀 양식 수집 및 항목별 전산화 가능성 검토. RPA 도입 시 EDI 수집 자동화 ROI 분석.
* 매장 운영 및 정보 관리
  - 질문: "매장 신규 등록 및 변경 시 부서별 업무 분담(영업팀/영업관리팀)은 어떻게 이루어지는가? 수기 입력 시 가장 잦은 오류는?"
  - 분석: 기능별 탭(Tab) 분리 및 권한 기반의 정보 등록 프로세스 설계. 매장 ID 자동 채번 기능 연동.
* 특수 재고 및 LOSS 처리
  - 질문: "매장 재고 실사 차이(Loss) 발생 시 전산 처리의 번거로운 점은? 창고 반품 없이 현장에서 즉시 처리해야 할 케이스는?"
  - 분석: 실물 이동 없는 전산 재고 조정(Adjustment) 프로세스 표준화 및 증빙 관리 방안.

[도출해야 할 결론 (To-Be)]
- 수기 정산 항목의 100% 전산화 및 RPA를 통한 EDI 데이터 연동 자동화.
- 매장 운영 업무 절차 간소화 및 오입력 방지를 위한 입력 자동화(Auto-fill).
- 현장 중심의 실시간 재고 조정 프로세스 확립 및 투명한 LOSS 처리 체계 구축.

3. 분석 결과 정리 프레임워크 (To-Be Matrix)

인터뷰 후 아래 양식으로 결론을 도출하여 차세대 설계의 근거로 활용합니다.

- 영역: [마스터 / 영업관리]
- 현행 이슈 (Pain Point): 구체적인 수치나 사례 (예: 매장 정보 수기 입력 오류 월 5건 등)
- 개선 방향 (To-Be Solution): 시스템 기능 및 프로세스 개선안
- 기대 효과 (Value): 시간 단축, 비용 절감, 데이터 무결성 등

4. 인터뷰 및 수기 문서 수집 이후 진행 프로세스 (Detailed Design)

현업의 목소리(인터뷰)와 실제 데이터(수기 문서)를 바탕으로 시스템 설계로 전환하는 단계입니다.

* 1단계: 수기 문서 데이터 구조 분석 (Reverse Engineering)
  - 확보한 엑셀/장부의 컬럼(Column)을 시스템 필드로 변환.
  - 엑셀 내 계산 수식/매크로를 파악하여 시스템 자동 로직으로 도출.
* 2단계: AS-IS vs To-Be Gap 분석
  - "왜 시스템이 아닌 엑셀을 썼는가?"에 대한 근본 원인 해결 전략 수립.
  - 수기로 진행되던 중복 프로세스 제거 및 최적화(Optimization).
* 3단계: To-Be 프로세스 맵 설계
  - 마스터 정보 생성 → 영업 활용 → 정산 → 회계 전송의 End-to-End 흐름도 설계.
  - 유통망 EDI 데이터가 시스템에 자동 안착되는 인터페이스 흐름 정의.
* 4단계: 데이터 모델링 및 가버넌스 수립
  - 분석된 항목을 바탕으로 DB 테이블 스키마 설계 (2자리 브랜드 코드 등 반영).
  - 분산된 수기 명칭을 표준 코드로 매핑하는 데이터 표준화 작업 수행.
* 5단계: 화면 설계(Wireframe) 및 현업 검증(Prototyping)
  - 엑셀의 편리함(대량 입력 등)을 시스템 UI/UX로 구현.
  - 프로토타입 리뷰를 통해 요구사항 누락 방지 및 최종 확답 확보.

[부록] 영역별 심층 인터뷰 질문 리스트 (Questionnaire)

1. 마스터 정보 (Master Data) 부문
- 브랜드 코드: "현재 잔여 코드가 4개뿐인데, 신규 브랜드 론칭 시 코드를 재사용하거나 '돌려막기' 했던 사례가 있습니까? 코드 확장 시 POS 출력물이나 택(Tag) 디자인에 물리적 제약이 발생합니까?"
- 칼라/사이즈: "신규 칼라 등록 시 기존 코드를 검색하지 않고 새로 생성하게 되는 결정적인 이유는 무엇입니까? (UI 불편함, 기준 모호 등) 브랜드별 기획 사이즈 그룹 이원화가 생산 관리에 실질적 도움이 됩니까?"
- 매장 마스터: "매장 ID 생성 규칙을 무시하고 임의로 생성해야만 했던 예외 케이스가 있습니까? 매장 정보 변경(점주 교체 등) 이력 부재로 인해 정산 시 곤란했던 사례가 있습니까?"

2. 영업관리 (Settlement & Operation) 부문
- 정산 자동화: "엑셀로 관리하는 공제 항목(수선비, 부자재비 등) 전산화 시, 항목별 승인 프로세스가 필요합니까? RPA 도입 시 어느 유통망의 EDI 데이터를 가장 먼저 자동화하길 원하십니까?"
- 업무 권한: "영업팀이 매장 정보를 직접 등록한다면, 영업관리팀은 어떤 핵심 항목에 대해서만 최종 검수(Confirm) 권한을 가지고 싶으십니까?"
- 재고/LOSS 처리: "매장 LOSS 처리를 위해 재고를 창고로 '가상 반품' 시키는 행정 작업에 소요되는 시간은 어느 정도입니까? 현장에서 즉시 LOSS 처리를 승인할 수 있는 가이드라인(전결권)이 필요합니까?"
- 수기 업무: "현재 시스템이 지원하지 않아 어쩔 수 없이 엑셀로 관리 중인 '가장 복잡한 양식'은 무엇입니까? (파일 샘플 확보 필요)"

※ 참고: 분석 시 현업이 사용하는 '실제 엑셀 파일'과 '수기 장부' 샘플을 반드시 확보하여 데이터 구조를 파악할 것.
